# Vassal Flatpak

This is the repository of the [Vassal](https://vassalengine.org) Flatpak package.


## How to make a new release

The recipe files [`org.vassalengine.vassal.yml`](org.vassalengine.vassal.yml) and [`maven-dependencies.yml`](maven-dependencies.yml) are generated by the regular Vassal GitHub [`release.yml`](https://github.com/vassalengine/vassal/blob/master/.github/workflows/release.yml) workflow, and stored as the artefact `flatpak-recipe` from the workflow.   This ensures that everything is up-to-date in the YAML files.

When the Vassal GitHub `release.yml` workflow has run for release _x.y.z_, then one can update this repository (or a fork thereof) by running the script [`update_from_release.sh`](update_from_release.sh) like:

```
$ ./update_from_release.sh --release x.y.z --token TOKEN
```
where _x.y.z_ is the newly released release number of Vassal, and _TOKEN_ is either a GitHub personal access token or a file that contains such a token.

The update should then, once the script has run,

- Possibly make a branch, say
  ```
  $ git checkout -b update-to-x.y.z
  ```
- Commit the changes
  ```
  $ git add maven-dependencies.yml  org.vassalengine.vassal.yml
  $ git commit -m "Update to release x.y.z"
  ```
- Push the commit (possibly to the fork)
  ```
  $ git push
  ```
- Open a pull request against the parent repository.
